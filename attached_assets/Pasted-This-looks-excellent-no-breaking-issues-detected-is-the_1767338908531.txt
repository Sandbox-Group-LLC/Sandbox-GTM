Answers to clarifications below:

1) Momentum Score Point Values (Tier 2)

Use the following Tier 2 (momentum modifier) points:

- intent_level:
  - LOW = +1
  - MEDIUM = +2
  - HIGH = +3

- outcome = intro_to_stakeholder: +2
- tag = decision_maker: +1
- tag = executive: +1
- each additional product interaction after the first: +1

- opportunityPotential (MAX bucket only per contact; do not sum):
  - <10k = +1
  - 10k–50k = +2
  - 50k–100k = +3
  - 100k+ = +3 (cap here to avoid inflation)

Rationale: opportunity size should contribute to momentum, but never overpower explicit intent.

---

2) “two+ interactions with intent_level >= MEDIUM”

This means:
- **two or more PRODUCT_INTERACTIONS only**
- intentLevel ∈ {medium, high}

Meetings are excluded from this rule because they already have Tier 1 triggers (e.g., active_opportunity, timeline=now) and shouldn’t double-count.

---

3) contra_signals `weight` field

Default to:
- `weight = "local_only"` for all `not_a_fit` outcomes.

Include a note like:
“Does not disqualify contact overall; indicates mismatch with this specific demo.”

Do not introduce any global disqualification logic here.
